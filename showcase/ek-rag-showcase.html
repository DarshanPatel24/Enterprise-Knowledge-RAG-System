Under the hood

Each stage is an independently deployable service. No engine reads another's database — they collaborate only via versioned REST + events and shared Pydantic contracts (packages/contracts). This decoupling lets any stage scale or be replaced without destabilizing the rest.

04 / HOW End-to-end in 60 seconds

From a raw file to a cited answer.

Convert. EKDC watches a folder and turns any dropped file — PDF, Word, PowerPoint, image, audio, video — into clean Markdown (with OCR & transcription).
Ingest. EKIE chunks the Markdown, creates embeddings, and publishes vectors into the database — each tagged with tenant, classification, and source path.
Ask. A user asks a question in the web chat; the request streams to EKCP with their identity and clearance.
Retrieve. EKCP asks EKRE, which searches, fuses, and re-ranks the best passages — filtered by the user's clearance before anything is ranked.
Answer. EKCP grounds the language model on those passages and streams the answer back token-by-token, with citation cards for every source used.
Under the hood

The browser consumes Server-Sent Events: stage frames (understanding → retrieving → reranking → reasoning → generating), then token and citation frames. If retrieval is slow or returns HTTP 429, a circuit breaker degrades gracefully to memory instead of failing the chat.

08 / UNDER THE HOOD Stack & models

A modern, local-first stack — no cloud dependencies.

Experience
Next.js 16 · React 19 · TypeScript · Tailwind + shadcn/ui — streaming chat, citations, clearance badges.
Services
Python 3.11+ · FastAPI · Pydantic v2 · SQLAlchemy — four decoupled engines behind versioned contracts.
Orchestration
LangChain · LangGraph — behind engine-owned abstractions so the core stays model-independent.
Models
Embeddings BAAI/bge-base-en-v1.5 (768-dim, cosine) · Reranker bge-reranker-base · Chat Qwen2.5-3B-Instruct · Image captions Qwen2.5-VL-3B.
Data & infra
Qdrant · Redis · MinIO · MS SQL Server — all self-hosted, run locally.
Observability
Langfuse · OpenTelemetry — structured JSON logs carry tenant & correlation IDs.
In plain English

Everything runs on hardware you control. Models are swappable — nothing is locked to a single vendor, and no data goes to the cloud.

09 / UNDER THE HOOD Trust & governance

The features that make it enterprise-grade.

Citations by default — if an answer can't be cited, it isn't generated.
Clearance-aware retrieval — filtered at the database boundary, before ranking.
PII masking — emails, SSNs, cards, phones scrubbed before answers leave.
Signed security context — verifiable trust between engines (JWT/HMAC).
DSAR / GDPR purge — "delete my data" fans out across engines.
Audit trails — every grant, denial, and purge is recorded.
Graceful degradation — a slow or down dependency never crashes the chat.
Multi-tenant isolation — one tenant's data is never returned to another.
